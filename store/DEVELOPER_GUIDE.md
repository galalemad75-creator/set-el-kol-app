# Developer Guide — ست الكل | Set El Kol

## Prerequisites

- Node.js 18+ and npm
- For Android: Android Studio, Java 17+, Android SDK 34
- For iOS: macOS, Xcode 15+, CocoaPods

## Quick Start

```bash
cd set-el-kol-app

# 1. Install dependencies
npm install

# 2. Add native platforms
npx cap add android
npx cap add ios

# 3. Sync web assets to native projects
npx cap sync

# 4. Open in IDE
npx cap open android   # opens Android Studio
npx cap open ios       # opens Xcode
```

## Building for Android

### 1. Add Android Platform
```bash
npx cap add android
npx cap sync
```

### 2. Open in Android Studio
```bash
npx cap open android
```

### 3. Configure Signing

#### Generate a Keystore (first time only):
```bash
keytool -genkey -v -keystore setelkol-release.keystore \
  -alias setelkol -keyalg RSA -keysize 2048 -validity 10000
```

#### Configure in Android Studio:
1. Go to **Build → Generate Signed Bundle/APK**
2. Select **Android App Bundle** (AAB) for Play Store, or **APK** for direct install
3. Click **Create new...** keystore
4. Fill in the keystore path, alias, and passwords
5. Select **release** build variant
6. Click **Finish**

#### Or configure in `android/app/build.gradle`:
```gradle
android {
    signingConfigs {
        release {
            storeFile file("../../setelkol-release.keystore")
            storePassword "YOUR_STORE_PASSWORD"
            keyAlias "setelkol"
            keyPassword "YOUR_KEY_PASSWORD"
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 4. Build APK/AAB

```bash
cd android

# Debug APK
./gradlew assembleDebug

# Release APK
./gradlew assembleRelease

# Release AAB (for Play Store)
./gradlew bundleRelease
```

Output locations:
- APK: `android/app/build/outputs/apk/release/app-release.apk`
- AAB: `android/app/build/outputs/bundle/release/app-release.aab`

### 5. Submit to Google Play Store

1. Go to [Google Play Console](https://play.google.com/console)
2. Create a new app (one-time $25 fee)
3. Fill in store listing (see `store/play-store-listing.md`)
4. Upload the AAB under **Release → Production**
5. Complete content rating questionnaire
6. Set up pricing (Free)
7. Submit for review

## Building for iOS

### 1. Add iOS Platform
```bash
npx cap add ios
npx cap sync
```

### 2. Open in Xcode
```bash
npx cap open ios
```

### 3. Configure Signing
1. In Xcode, select the **App** target
2. Go to **Signing & Capabilities**
3. Select your **Team** (Apple Developer account)
4. Set **Bundle Identifier**: `com.setelkol.recipes`
5. Ensure "Automatically manage signing" is checked

### 4. Build & Archive
1. Select **Any iOS Device (arm64)** as target
2. Go to **Product → Archive**
3. Once archived, click **Distribute App**
4. Select **App Store Connect**
5. Follow the prompts to upload

### 5. Submit to App Store Connect

1. Go to [App Store Connect](https://appstoreconnect.apple.com)
2. Create a new app ($99/year Apple Developer Program required)
3. Fill in app information (see `store/app-store-listing.md`)
4. Upload screenshots for required device sizes
5. Fill in privacy policy URL
6. Set pricing (Free)
7. Submit for review

## Common Commands

```bash
# Sync web assets to native
npx cap sync

# Sync and open Android
npx cap sync android && npx cap open android

# Sync and open iOS
npx cap sync ios && npx cap open ios

# Update Capacitor
npm install @capacitor/core@latest @capacitor/cli@latest

# Run on device (Android)
npx cap run android --target <device-id>

# Run on device (iOS)
npx cap run ios --target <device-id>
```

## Updating the App

1. Make changes to files in `www/`
2. Run `npx cap sync` to copy changes to native projects
3. Build and test in the IDE
4. Submit update to app stores

## Troubleshooting

### Android build fails
- Ensure Java 17 is installed: `java -version`
- Ensure Android SDK 34 is installed via SDK Manager
- Clean build: `cd android && ./gradlew clean`

### iOS build fails
- Run `cd ios/App && pod install`
- Clean build folder in Xcode: **Product → Clean Build Folder**
- Ensure Xcode Command Line Tools are installed: `xcode-select --install`

### Capacitor sync issues
- Delete `android/` and `ios/` folders
- Run `npx cap add android` and `npx cap add ios` again
- Run `npx cap sync`

### Assets not loading on device
- Ensure all asset paths are relative (no leading `/` unless needed)
- Run `npx cap sync` after any file changes
- Check that assets exist in `www/` before syncing
